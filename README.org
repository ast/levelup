A Rust workspace of personal-computing tools.

* Crates

** hugin

A Wayland clipboard manager for sway and other wlroots-based compositors. Watches the clipboard via =wlr-data-control-unstable-v1= and persists every selection change to SQLite. Both the *regular* selection (Ctrl-C / Ctrl-V) and the *primary* selection (mouse text selection / middle-click) are tracked as independent streams. Honours the =x-kde-passwordManagerHint=secret= convention used by KeePassXC, Bitwarden, and 1Password to keep passwords out of history.

Planned: expose history to AI via MCP; synchronize across machines.

** munin

An atuin-like shell history search with an fzf-style TUI. Currently a stub.

* Requirements

- Rust 1.85+ (edition 2024)
- A wlroots-based Wayland compositor (sway, hyprland, river). =wlr-data-control-unstable-v1= is *not* implemented by GNOME or KDE.
- =wl-clipboard= for the smoke-test recipe (=wl-copy=, =wl-paste=).
- =just= (optional, for the workspace recipes — see [[file:justfile][justfile]]).

* Running hugin

** Daemon (=hugind=)

A foreground process intended to be supervised by =systemd --user= eventually; run it directly from a shell during development.

#+begin_src sh
# Default DB at $XDG_DATA_HOME/hugin/hugin.db,
# socket at $XDG_RUNTIME_DIR/hugin.sock
cargo run --bin hugind -p hugin

# Override paths and crank up logging
HUGIN_LOG=debug ./target/release/hugind \
    --db /tmp/hugin.db --socket /tmp/hugin.sock
#+end_src

See =hugind --help= for retention flags (=--max-entries=, =--max-age-days=).

** CLI (=hugin=)

The =hugin= CLI talks to the running daemon over its unix socket. By default it connects to =$XDG_RUNTIME_DIR/hugin.sock=; pass =--socket PATH= to override.

| Subcommand                                     | What it does                                                  |
|------------------------------------------------+---------------------------------------------------------------|
| =hugin ping=                                   | Verify the daemon is responsive.                              |
| =hugin list [--limit N] [--selection regular\|primary]= | Print recent entries, newest first.                  |
| =hugin info ID=                                | Show metadata for one entry (timestamp, selection, MIMEs).    |
| =hugin get ID [--mime MIME]=                   | Write the entry's contents to stdout. Default MIME is the first =text/*= advertised, falling back to the first available. |
| =hugin copy ID [--selection regular\|primary]= | Put an old entry back on the clipboard. The daemon becomes the data source until another app takes over. |

Each subcommand has a short unix-style alias: =ls= (list), =stat= (info), =cat= (get), =cp= (copy), =comp= (completions). =hugin --help= shows them.

Examples:

#+begin_src sh
# What's in the recent history?
hugin list

# Just things selected with the mouse (primary), last 10
hugin list --selection primary --limit 10

# Inspect which MIME variants exist for an entry
hugin info 42

# Get the plain text…
hugin get 42

# …or the HTML the source app also offered
hugin get 42 --mime text/html

# Put an old entry back on the clipboard (daemon becomes the source)
hugin copy 42

# Same, but onto the primary selection
hugin copy 42 --selection primary
#+end_src

*** Shell completions

#+begin_src sh
# zsh
mkdir -p ~/.config/zsh/completions
just completions-hugin  zsh > ~/.config/zsh/completions/_hugin
just completions-hugind zsh > ~/.config/zsh/completions/_hugind
# add `fpath+=(~/.config/zsh/completions)` to ~/.zshrc before `compinit`

# fish
just completions-hugin  fish > ~/.config/fish/completions/hugin.fish
just completions-hugind fish > ~/.config/fish/completions/hugind.fish

# bash
just completions-hugin  bash > ~/.local/share/bash-completion/completions/hugin
just completions-hugind bash > ~/.local/share/bash-completion/completions/hugind
#+end_src

The binaries also know how to produce their own completions: =hugin completions zsh= and =hugind --generate-completions zsh= are equivalent to the just recipes.

** Running under systemd

For day-to-day use, run =hugind= as a user service tied to the graphical session. A ready-to-edit unit lives at [[file:dist/hugind.service][=dist/hugind.service=]] (it expects the binary at =~/.cargo/bin/hugind=, i.e. wherever =just install= puts it).

#+begin_src sh
just install
mkdir -p ~/.config/systemd/user
cp dist/hugind.service ~/.config/systemd/user/
systemctl --user daemon-reload
systemctl --user enable --now hugind.service
journalctl --user -u hugind.service -f                     # tail logs
#+end_src

Sway needs to publish =WAYLAND_DISPLAY= into the user manager and activate the target. Add to =~/.config/sway/config=:

#+begin_src sh
exec systemctl --user import-environment WAYLAND_DISPLAY XDG_CURRENT_DESKTOP
exec systemctl --user start graphical-session.target
#+end_src

The unit uses =Type=notify=, so =systemctl --user start hugind= blocks until the daemon is ready. =PartOf=graphical-session.target= stops hugin when sway exits; =Restart=on-failure= retries transient errors (e.g. compositor restart) without thrashing.

To pass flags (=--primary= for instance), override the unit:

#+begin_src sh
systemctl --user edit hugind.service
# In the editor:
# [Service]
# ExecStart=
# ExecStart=%h/.cargo/bin/hugind --primary
#+end_src
